Task 1: Extend schema and backend for lot-based compliance traceability

First inspect:
- shared schema definitions
- drizzle config and migrations
- server storage/repository layer
- server routes/controllers related to lots, batches, stock movements, and traceability

Goal:
Add backend support for full lot-based receiving, barcode values, lot-based consumption, finished lot creation, and deeper traceability.

Requirements:

1. Extend the lots model
Add or confirm fields for:
- lotType: raw_material | intermediate | finished_good
- barcodeValue
- status: active | quarantined | released | consumed | expired
- sourceType: supplier | farmer | internal_batch
- sourceName
- supplierLot
- receivedDate
- producedDate
- expiryDate
- originalQuantity
- remainingQuantity
- location or locationId
- barcodePrintedAt
- notes

2. Add goods received backend flow
Create backend support for receiving raw stock.
This action must:
- create a lot
- generate a unique internal lot number
- generate a barcode value
- create a stock movement record
- increase inventory correctly

3. Enforce lot-based production consumption
Batch inputs must consume exact lots, not generic items only.
Consumption must:
- validate lot exists
- validate sufficient remaining quantity
- reduce remaining quantity
- create stock movement records
- preserve linkage between batch and consumed lot

4. Create finished output lots on batch finalization
When a batch is completed/finalized:
- create one or more finished lots
- assign lot number and barcode value
- record produced date and quantity
- link each finished lot to the source batch
- create stock movement records

5. Expand traceability queries
Support querying:
- all input lots for a batch
- all output lots for a batch
- all batches that consumed a lot
- full backward and forward lot lineage

6. Auditability
Ensure receiving, consuming, and finalizing actions are auditable.

Deliverables:
- schema changes
- migration(s)
- storage layer updates
- route/controller updates
- helper utilities for lot number generation and barcode value generation

Acceptance criteria:
- raw stock can be received as a lot
- lots have barcode values
- production consumes exact lots
- finalizing a batch creates finished lots
- traceability can move backward and forward across lots and batches
